Contexte technique

Nature et fonctionnalités du projet

NetMonitor est un système client-serveur permettant de :

  • Surveiller en temps réel les ressources système (CPU, mémoire, disque, réseau) de plusieurs machines sur un réseau local
  • Partager des fichiers entre les machines connectées
  • Visualiser les données via une interface web interactive

Le projet est développé avec les technologies suivantes :

Backend : Python avec architecture modulaire et programmation orientée objet

Communication réseau : Sockets TCP/IP, sélection non-bloquante (select())

API Web : Flask pour exposer une API REST et servir l'interface web

Frontend : HTML/CSS/JavaScript avec Chart.js pour les visualisations

Collecte de données : Module psutil pour obtenir les métriques système

L'application est fournie dans un contexte éducatif et n'est pas destinée à un usage en production.

Protection des données

Informations sur les données collectées

Important : NetMonitor est un projet étudiant développé dans un cadre pédagogique. Les données collectées sont utilisées uniquement à des fins de test et de démonstration.

Données collectées

Dans le cadre de ce projet, l'application collecte :

  • Des métriques système (CPU, mémoire, disque, réseau)
  • Des noms d'hôtes et adresses IP pour la démonstration
  • Des informations sur les fichiers partagés entre machines

Finalité pédagogique

Ces données sont utilisées uniquement pour :

  • Démontrer les fonctionnalités du projet
  • Évaluer le projet dans le cadre académique
  • Tester les capacités techniques du monitoring réseau

Ce projet n'est pas destiné à être utilisé en production avec des données réelles sensibles. En cas d'utilisation en dehors du cadre académique, il conviendrait d'effectuer une mise en conformité complète avec le RGPD.
